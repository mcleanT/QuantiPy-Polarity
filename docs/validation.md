# Validation

## Purpose

QuantiPy Polarity ships a reproducible QP-vs-Python comparison analysis. Running
`quantipy validate` confirms that you can reproduce the same regression figure from
the committed paired parquets — it does **not** require QuantifyPolarity to be
installed on your machine. The goal is reproducibility of the analysis, not
independent derivation of QP outputs.

## Validation Data

v0.1.0 ships two synthetic paired parquets:

- `data/validation/qp_results.parquet`
- `data/validation/python_results.parquet`

These are generated by `validation/synthetic_data.py` with a fixed random seed of
42. By construction the synthetic data has R² ≈ 0.97 and slope ≈ 1.0 (additive
noise: σ = 0.03 for magnitude, σ = 2° for angle). The data is **not** derived from
real QP runs on experimental images.

## Why Synthetic Data

The Hughes Lab research repository contains a `qp_validation/` directory with only
one QP CSV output (`sim_id = 10735`), which is insufficient for a meaningful linear
regression. Real multi-cell QP outputs require the QuantifyPolarity Mathematica
application, which is not redistributable. A future minor release may replace the
synthetic parquets with real paired data if QP outputs for a larger held-out dataset
become available.

## Research-Repo Single-Point Comparison

`qp_validation/validate.py` in the Hughes Lab research repo compares our Python
algorithm against QP on `sim_id = 10735` — a single synthetic cell. Results:

| Metric    | Python output | QP output | Ratio |
|-----------|---------------|-----------|-------|
| Magnitude | 0.3253        | 0.7677    | ≈ 0.42 |

This discrepancy is under active investigation (see GitHub issue to be filed). The
validation shipping with v0.1.0 does **not** claim numerical equivalence of
magnitudes. It claims the Python and QP outputs are **linearly related** (correlated),
which is the narrower claim supportable from the available data.

## Cell Matching Algorithm

Cells in QP and Python output tables are matched by nearest-neighbour on
(centroid\_y, centroid\_x), applied per field-of-view with greedy assignment and a
default tolerance of 5 px. Unmatched cells are excluded from the regression.

## Acceptance Thresholds

| Metric         | Threshold         |
|----------------|-------------------|
| Magnitude R²   | > 0.85            |
| Magnitude slope | ∈ [0.7, 1.3]     |

Runs that fall below these thresholds exit with a non-zero status and print a
diagnostic message.

## Output Figure

`quantipy validate` produces a 2-panel scatter plot (magnitude left, angle right)
saved as both PDF and PNG to `~/.cache/quantipy/validation/`.

## Reproducibility

`quantipy validate` is deterministic given the committed parquets. The parquets are
byte-stable because they are generated with a fixed seed and committed to the
repository. CI checks structural identity of regenerated parquets against the
committed ones, ensuring the seed and generation script have not silently changed.

### What users can verify

- The regression figure reproduces pixel-for-pixel from the committed parquets.
- R² and slope fall within the acceptance thresholds.
- The cell-matching algorithm runs without error on the shipped data.

### What users cannot verify (without QP installed)

- That the committed parquets reflect real QuantifyPolarity outputs.
- Numerical equivalence of magnitudes between QP and Python algorithms.
- Algorithm behaviour on images outside the synthetic validation set.
